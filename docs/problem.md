# 문제 정의 (Problem Statement)

## 현재 상태 (AS-IS)

### 배합 의사결정 구조의 한계

```
[현재 배합 결정 프로세스]

성분 검사 결과 ──→ 담당자 (경험·감 의존) ──→ 배합비율 결정
(개별 문서 관리)                              (비표준화)
```

| 문제 영역 | 세부 내용 |
|-----------|-----------|
| 데이터 단절 | 성분분석 데이터가 개별 문서로 관리되어 공정 데이터와 연계 불가 |
| 경험 의존 | 배합비율이 작업자 숙련도에 따라 결정되어 재현성 부족 |
| 사전 보정 불가 | 원재료 LOT별 성분 편차에 대한 사전 보정 체계 부재 |
| 불량 원인 불투명 | 품질 이상 발생 시 원인 추적에 긴 시간 소요 |
| 패턴 분석 부재 | 과거 LOT 성분 비교 및 패턴 분석 도구 없음 |

### 핵심 문제 정의

> **원재료 성분 편차 → 배합 결정 로직 미흡 → 품질 변동 → 부적합률 누적**

이 인과 사슬에서 **배합 결정 로직**을 데이터 기반으로 전환하는 것이 본 프로젝트의 핵심 개입 지점이다.

---

## 목표 상태 (TO-BE)

```
[목표 배합 결정 프로세스]

성분 검사 결과 ──→ ML 모델 ──→ 최적 배합비율 자동 추천
(통합 DB)        (XGBoost/RF)   (작업자 확인 후 실행)
     ↑
공정 조건 + 품질 이력 (피드백 루프)
```

### 해결 과제 분해

| # | 과제 | 해결 방법 |
|---|------|-----------|
| 1 | 성분-품질 상관관계 불명확 | 성분값 × 공정조건 → 품질 예측 회귀 모델 |
| 2 | 최적 배합비율 도출 불가 | 품질 예측 모델 기반 배합비율 역추적(최적화) |
| 3 | 원재료 편차 대응 | LOT별 성분 편차를 피처로 반영한 실시간 추천 |
| 4 | 불량 사전 예방 | 배합 전 품질 예측값이 임계치 하회 시 경고 |

---

## 머신러닝 문제 정의

### Task 1 — 품질 예측 (핵심)

```
Input:  성분값[Sn, Ag, Cu, ...], 배합비율, 용해온도, 가열시간, 원재료 LOT편차
Output: 품질등급 (연속값) 또는 불량 여부 (이진)
Type:   Regression (품질 점수) / Classification (불량 여부)
```

### Task 2 — 최적 배합비율 추천

```
Input:  현재 원재료 성분 분석값 (신규 LOT)
Output: 품질 최적화 조건에서의 배합비율 추천값
Type:   Optimization (Task 1 모델을 objective function으로 활용)
```

## 데이터 요구사항

### 필요 데이터 (수집 대상)

| 데이터 유형 | 변수 예시 | 수집 방법 |
|-------------|-----------|-----------|
| 성분 분석값 | Sn(%), Ag(%), Cu(%), Pb(%), 기타 | XRF 분석기 |
| 배합비율 | 각 성분 투입 비율 | 작업 기록 |
| 공정 조건 | 용해온도, 가열시간, 냉각조건 | PLC/IoT 센서 |
| 품질 검사 | 불량유형, 품질등급, 합격여부 | 검사 기록 |
| 원재료 메타 | 공급사, LOT번호, 입고일 | 바코드/입고 이력 |

### 데이터 품질 기준

- 최소 학습 데이터: 배치 단위 **300건 이상** 권장
- 품질 라벨: 연속값(품질점수) 또는 이진(합격/불합격) 중 하나 확정 필요
- 결측치: 성분값 결측 시 보간 불가 — 해당 LOT 제외 원칙
