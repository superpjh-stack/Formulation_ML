## 2. 현황 및 문제

### 현재 운영 방식

- 성분분석(XRF) 데이터는 수집되나 **개별 문서로 관리**되어 공정 데이터와 연계 불가
- 배합비율이 **작업자 경험에 의존**하여 표준화 부재
- 원재료 LOT별 편차에 대한 **사전 보정 체계 없음**
- 배치별 품질 편차 발생 시 **원인 추적 어려움**

### 핵심 문제

```
원재료 성분 편차
    → 비표준화된 배합 결정 (경험 의존)
        → 배치별 품질 변동
            → 부적합률 누적 및 재작업 비용 발생
```

## 3. 목표 및 성과지표 (KPI)

| KPI | 현재 수준 | 목표 수준 | 측정 방법 |
|-----|-----------|-----------|-----------|
| 부적합률 | 기준값 (측정 후 확정) | 20~30% 감소 | 월별 불량 건수 / 전체 생산 LOT |
| 모델 R² | — | ≥ 0.85 | 테스트셋 평가 |
| 모델 MAPE | — | ≤ 10% | 테스트셋 평가 |
| 배합 추천 소요시간 | 작업자 판단 (수분) | < 30초 (자동) | 추론 응답 시간 |

## 4. 솔루션 구성

### 핵심 AI 모델

```
[입력]                          [모델]                  [출력]
성분분석값 (Sn, Ag, Cu...)  ┐
용해온도, 가열시간           ├─→  XGBoost/Random Forest ─→  품질 예측값
원재료 LOT 편차             ┘                               최적 배합비율 추천
```

### 시스템 구성

```
data/raw/          ← XRF 성분 분석 CSV, 공정 기록, 품질 검사 결과
    ↓ (전처리)
data/processed/    ← 정제된 피처 데이터
    ↓ (학습)
models/artifacts/  ← 학습된 모델 (.joblib)
    ↓ (추론)
배합비율 추천 결과 (data/processed/recommendations.csv)
```

### 기술 스택

| 구분 | 기술 |
|------|------|
| 언어 | Python 3.10+ |
| 핵심 라이브러리 | scikit-learn, pandas, numpy, scipy |
| 목표 모델 | XGBoost (xgboost>=1.7) |
| 최적화 | scipy.optimize.minimize (SLSQP) |
| 모델 저장 | joblib |

## 5. 개발 단계 (Phases)

### Phase 1 — 데이터 수집 및 탐색 (EDA)
- [ ] 과거 성분분석 데이터 수집 (최소 300 LOT 이상)
- [ ] 공정 조건 데이터 수집 (온도, 시간)
- [ ] 품질 검사 결과 수집 (불량유형, 합격여부)
- [ ] `notebooks/01_eda.ipynb` 에서 분포, 상관관계, 이상치 분석
- [ ] 피처 후보 선정 및 타겟 변수 확정

### Phase 2 — 피처 엔지니어링 및 베이스라인
- [ ] `src/features/engineering.py` 피처 로직 구체화
- [ ] Ridge, Random Forest 베이스라인 학습 및 CV 성능 확인
- [ ] 피처 중요도 분석 → 불필요 변수 제거

### Phase 3 — 모델 고도화
- [ ] XGBoost 모델 추가 (`requirements.txt`에 `xgboost` 추가)
- [ ] 하이퍼파라미터 튜닝 (GridSearchCV / Optuna)
- [ ] RMSE, R², MAPE 기준 달성 여부 검증

### Phase 4 — 배합비율 최적화 구현
- [ ] `scipy.optimize` 기반 최적 배합 역추적 로직 구현
- [ ] 성분 합계 = 100% 제약조건 반영
- [ ] 추천 결과 검토 (도메인 전문가 검증)

### Phase 5 — 운영 체계 구축
- [ ] `scripts/predict.py` — 신규 LOT 자동 추천 파이프라인 완성
- [ ] 성능 모니터링 지표 정의 (주간 R² 추적)
- [ ] Online Learning 전략 수립 (warm_start 또는 full retraining 주기)

## 7. 의사결정 필요 사항

다음 항목은 도메인 전문가 또는 이해관계자와 확정 필요:

4. **추천 UI/UX** — CSV 파일 출력 vs 대시보드 vs 현장 단말(스마트패드) 연동
