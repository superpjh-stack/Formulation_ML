## 1. 시스템 개요

### 1.1 시스템 소개

Formulation ML은 납땜 합금(솔더) 배합 공정의 SN(주석)/AG(은)/CU(구리)/PB(납) 성분 배합비율을 ML 알고리즘으로 자동 최적화하는 스마트 제조 시스템입니다.

기존에 숙련 작업자의 경험에만 의존하던 배합비율 결정 과정을 데이터 기반으로 전환하여, 누구나 최적 배합비율을 3초 이내에 추천받고 품질 점수를 사전에 예측할 수 있습니다.

**주요 특징**:
- 300 LOT 이력 데이터 학습 기반 ML 예측 (GradientBoosting MAPE 2.78%)
- SLSQP 최적화 알고리즘으로 공정 조건 맞춤형 배합비율 자동 산출
- 공급사별 성분 편차 반영 (SUP_A/B/C)
- 6종 DOE 실험 설계 자동화 (Full Factorial, CCD, Box-Behnken 등)
- 웹 브라우저 기반 직관적 UI (별도 설치 불필요)

### 1.2 주요 기능 요약

| 기능 | 메뉴 경로 | 주요 용도 |
|------|-----------|-----------|
| 배합비율 추천 | /recommend | 공정 조건 입력 → 최적 SN/AG/CU/PB 비율 추천 |
| 품질 점수 예측 | /predict | 성분 비율 직접 입력 → 품질 점수 사전 예측 |
| 데이터 분석(EDA) | /eda | 성분 분포, 상관관계, 목표값 편차 시각화 |
| 모델 현황 | /models | MAE/RMSE/R²/MAPE 성능 지표 + 피처 중요도 |
| DOE 시뮬레이터 | /doe | 실험계획 설계 → ML 시뮬레이션 → 통계 분석 → 최적화 |

## 2. 시작하기

### 2.1 사전 준비 사항

시스템 운영자(IT팀)가 다음 절차로 서버를 구동합니다. 일반 사용자는 웹 브라우저 접속만 하면 됩니다.

**Step 1. Python 환경 준비**

```bash
# Python 3.10 이상 필요
python --version

# 패키지 설치
pip install -r requirements.txt
```

**Step 2. 샘플 데이터 생성** (실데이터가 없는 경우)

```bash
python data/raw/generate_sample.py
```

생성 결과: `data/raw/formulation_history.csv` (300 LOT 샘플 데이터)

**Step 3. ML 모델 학습**

```bash
# GradientBoosting 모델 학습 (권장)
python scripts/train.py --data formulation_history.csv --target quality_score --model gradient_boosting

# 전체 모델 학습 (선택)
python scripts/train.py --data formulation_history.csv --target quality_score --model random_forest
python scripts/train.py --data formulation_history.csv --target quality_score --model xgboost
python scripts/train.py --data formulation_history.csv --target quality_score --model ridge
```

학습 완료 시: `models/artifacts/gradient_boosting.joblib` 및 `models/artifacts/preprocessors_gradient_boosting.joblib` 생성

**Step 4. FastAPI 백엔드 서버 실행**

```bash
python -m uvicorn app:app --port 8000 --reload
```

정상 실행 확인: `http://localhost:8000/docs` 접속 시 Swagger UI 표시

**Step 5. Next.js 프론트엔드 실행**

```bash
cd frontend
npm install   # 최초 1회만
npm run dev
```

정상 실행 확인: `http://localhost:3000` 접속 시 메인 화면 표시

## 3. 기능별 사용법

### 3.1 배합비율 추천 기능

**접속**: 상단 메뉴 "배합추천" 클릭 또는 `http://localhost:3000/recommend`

**용도**: 공정 온도·시간·공급사 조건을 입력하면 최적 SN/AG/CU/PB 배합비율과 예측 품질 점수를 자동으로 추천합니다.

#### 단계별 사용법

**1단계. 예측 모델 선택**

드롭다운 메뉴에서 모델을 선택합니다.

| 모델명 | RMSE | R² | 추천 여부 |
|--------|------|-----|-----------|
| gradient_boosting | 3.05 | 0.627 | 권장 |
| random_forest | 3.21 | 0.588 | 사용 가능 |
| xgboost | 3.18 | 0.594 | 사용 가능 |
| ridge | 4.52 | 0.341 | 기준선 참고용 |

> 미학습 모델 선택 시 "해당 모델의 학습 파일을 찾을 수 없습니다. 관리자에게 문의하세요." 안내 메시지가 표시됩니다.

**2단계. 공정 온도 입력**

- 입력 범위: 200~320°C
- 기본값: 250°C
- SAC 계열(무연) 기준 권장 범위: 240~280°C
- 입력 예시: `250`

> 범위 초과 입력 시 "온도는 200~320°C 범위로 입력하세요." 경고 메시지 표시

**3단계. 가열 시간 입력**

- 입력 범위: 10~120분
- 기본값: 45분
- 입력 예시: `45`

**4단계. 공급사 선택**

| 공급사 코드 | 설명 | 비율 |
|-------------|------|------|
| SUP_A | A공급사 (기준) | 50% |
| SUP_B | B공급사 (기준 대비 상대 편차 반영) | 30% |
| SUP_C | C공급사 | 20% |

**5단계. "배합비율 추천" 버튼 클릭**

로딩 스피너 표시 후 평균 1~3초 내 결과 반환

**6단계. 결과 확인**

```
추천 배합비율:
  SN (주석): 62.3%
  AG (은):    3.1%
  CU (구리):  0.5%
  PB (납):   34.1%
  합계:      100.0%

예측 품질 점수: 87.4점
품질 등급: 양호 (75점 이상)
```

#### 결과 해석 방법

| 품질 점수 | 등급 | 의미 | 조치 |
|-----------|------|------|------|
| 90점 이상 | 우수 | 최적 배합 상태 | 즉시 적용 가능 |
| 75~89점 | 양호 | 정상 품질 범위 | 적용 가능 |
| 75점 미만 | 불량 위험 | 불량 발생 가능성 높음 | 조건 재조정 필요 |

- **목표값 대비 편차**: SN ±2%, AG ±0.5%, CU ±0.2% 이내가 최적
- **권장 적용 기준**: 예측 품질 점수 80점 이상 + 성분 합계 100% 확인 후 적용
- 추천 결과는 참고용이며, 최종 배합 결정은 작업자가 승인합니다.

### 3.2 품질 점수 예측 기능

**접속**: 상단 메뉴 "품질예측" 클릭 또는 `http://localhost:3000/predict`

**용도**: 특정 배합비율을 직접 입력하여 배합 실행 전 품질 점수를 사전 확인합니다.

#### 성분 비율 입력 규칙

- SN, AG, CU, PB, OTHER 각각 % 단위로 입력
- **합계가 반드시 100%** 여야 합니다.
- 합계가 100%가 되지 않으면 "성분 합계가 100%가 되어야 합니다. 현재 합계: XX%" 경고 표시
- 소수점 1자리까지 입력 가능 (예: 62.3)

**입력 예시**:
| 성분 | 입력값 |
|------|--------|
| SN (주석) | 62.0 |
| AG (은) | 3.0 |
| CU (구리) | 0.5 |
| PB (납) | 34.5 |
| OTHER | 0.0 |
| 합계 | 100.0 |

공정 조건(온도, 시간, 공급사)도 함께 입력 후 "품질 예측" 버튼 클릭

#### 결과 해석

- 예측 품질 점수와 함께 목표값(SN 62.0%, AG 3.0%, CU 0.5%) 대비 편차 표시
- 75점 미만 예측 시 빨간색 경고 강조 표시

### 3.3 데이터 분석 (EDA)

**접속**: 상단 메뉴 "데이터분석" 클릭 또는 `http://localhost:3000/eda`

**용도**: 학습 데이터의 성분 분포, 상관관계, 목표값 대비 편차를 시각화하여 데이터 품질과 공정 현황을 파악합니다.

#### 차트별 해석 방법

**1. 성분 분포 히스토그램**
- X축: 성분 비율(%), Y축: 빈도수
- 목표값(SN 62%, AG 3%, CU 0.5%)에 분포가 집중되어 있는지 확인
- 넓게 분산된 경우: 공급사 편차 또는 공정 불안정 의심

**2. 품질 점수 분포**
- 75점 기준선(빨간색 점선)을 기준으로 불량/정상 비율 확인
- 정상 분포 확인: 80~95점 구간에 집중 분포가 이상적

**3. 성분-품질 상관관계 산점도**
- 각 성분 비율과 품질 점수 간의 관계 파악
- 양의 상관: 성분 증가 시 품질 향상 / 음의 상관: 성분 증가 시 품질 저하

**4. 공급사별 성분 편차 박스플롯**
- SUP_A/B/C별 SN/AG/CU 편차 분포 비교
- 박스 폭이 넓을수록 공급사 편차 큼 → 보정 필요

**5. 목표값 대비 편차 시계열**
- X축: LOT 순서, Y축: 편차(%)
- 추세선이 0 근처를 유지하면 공정 안정
- 특정 구간에서 편차 증가 시 해당 기간 원인 분석 필요

### 3.4 모델 현황

**접속**: 상단 메뉴 "모델현황" 클릭 또는 `http://localhost:3000/models`

**용도**: 학습된 ML 모델들의 성능 지표를 비교하고, 어떤 피처가 품질에 영향을 미치는지 확인합니다.

#### 성능 지표 의미

| 지표 | 설명 | 좋은 값 | 주의 수준 |
|------|------|---------|-----------|
| **MAE** (평균절대오차) | 예측값과 실제값 평균 절대 차이 | 낮을수록 좋음 | 5.0 초과 |
| **RMSE** (평균제곱근오차) | 큰 오차에 민감한 오차 지표 | 낮을수록 좋음 | 5.0 초과 |
| **R²** (결정계수) | 모델이 분산을 설명하는 비율 (0~1) | 1에 가까울수록 좋음 | 0.5 미만 |
| **MAPE** (평균절대백분율오차) | % 단위 예측 오차 | 낮을수록 좋음 | 5% 초과 |

> R² < 0.85는 현재 샘플 데이터의 노이즈(σ=3) 때문입니다. 실제 공정 데이터로 재학습 시 성능 향상이 예상됩니다.

#### 모델 선택 기준

1. **일반 운영**: gradient_boosting 사용 (RMSE 3.05, MAPE 2.78% — 최우수)
2. **안정성 우선**: random_forest 사용 (과적합 위험 낮음)
3. **성능 비교 목적**: 모든 모델 동시 예측 후 비교

#### 피처 중요도 해석

상위 중요 피처 예시:
1. `sn_deviation` (SN 목표값 대비 편차) — 품질에 가장 큰 영향
2. `melt_temp_c` (용해 온도) — 공정 조건 중 핵심
3. `ag_deviation` (AG 편차)
4. `melt_time_min` (가열 시간)
5. `cu_deviation` (CU 편차)

중요도가 높은 피처일수록 해당 변수 관리가 품질 안정화에 중요합니다.

### 3.5 DOE 시뮬레이터

**접속**: 상단 메뉴 "DOE" 클릭 또는 `http://localhost:3000/doe`

**용도**: 실험계획법으로 실험 설계 후 ML 시뮬레이션으로 결과를 예측하고, 통계 분석을 통해 최적 공정 조건을 도출합니다.

DOE 시뮬레이터는 4개 탭으로 구성됩니다.

#### 탭 1: 설계 (Design)

**1. DOE 방법 선택**

| 방법 | 특징 | 권장 사용 상황 |
|------|------|----------------|
| Full Factorial | 모든 조합 실험 | 인자 2~3개, 철저한 분석 |
| Fractional Factorial | 일부 조합으로 주효과 분석 | 인자 4개 이상, 스크리닝 |
| CCD (Central Composite) | 2차 반응표면 분석 | 비선형 관계 모델링 |
| Box-Behnken | CCD 대비 극단점 제외 | 실험 공간 제약 있을 때 |
| Taguchi | 잡음 인자 최소화 | 강건 설계 목적 |
| LHS (Latin Hypercube) | 균일한 공간 탐색 | 넓은 인자 범위 탐색 |

**2. 인자(Factor) 설정**
- 인자명 입력 (예: melt_temp_c, sn_pct)
- 최솟값/최댓값/수준 수 입력
- "인자 추가" 버튼으로 최대 5개까지 추가 가능

**3. 설계 행렬 생성**
- "설계 생성" 버튼 클릭 → 실험 조합 테이블 표시
- 각 행이 1개 실험 조합
- "CSV 다운로드" 버튼으로 저장 가능

#### 탭 2: 시뮬레이션 (Simulation)

**1. 공정 조건 설정**
- 고정 공정 조건 입력 (공급사, 기타 성분 비율)
- 설계 탭에서 생성한 실험 조합이 자동 반영

**2. ML 예측 실행**
- "시뮬레이션 실행" 버튼 클릭
- 각 실험 조합에 대해 ML 모델이 품질 점수 예측
- 실행 완료 후 결과 테이블 표시 (실험 조건 + 예측 품질 점수)

#### 탭 3: 분석 (Analysis)

**주효과(Main Effect) 해석**
- 각 인자 수준 변화에 따른 품질 점수 평균 변화량
- 기울기가 클수록 해당 인자의 영향력이 큼
- 그래프에서 가파른 기울기 인자 → 엄격한 관리 필요

**ANOVA 결과 해석**

| 열 | 의미 |
|----|------|
| Source | 인자명 |
| SS | 제곱합 (변동 크기) |
| df | 자유도 |
| MS | 평균제곱 |
| F-value | F 통계량 |
| p-value | 유의확률 (0.05 미만이면 유의미한 영향) |

p-value < 0.05인 인자: 품질에 통계적으로 유의미한 영향을 미치는 인자

#### 탭 4: 최적화 (Optimization)

**SLSQP 최적화**
- scipy SLSQP 알고리즘으로 품질 점수를 최대화하는 인자 조합 탐색
- 인자 범위 제약 조건 자동 반영
- 실행 시간: 수 초 이내
- 결과: 최적 인자 조합 + 예측 최대 품질 점수

**LHS 최적화 (비교용)**
- Latin Hypercube Sampling으로 넓은 공간 무작위 탐색
- SLSQP 대비 전역 최적해 탐색에 유리
- 실행 시간: 설정 샘플 수에 비례

**결과 해석**
- 두 방법의 결과를 비교하여 일치하면 강건한 최적해
- 불일치 시 여러 지역 최적해 존재 가능 → 추가 실험 권장

## 5. FAQ (자주 묻는 질문)

**Q1. 배합비율 추천 결과를 그대로 적용해야 하나요?**
A. 추천 결과는 참고용입니다. ML 모델의 예측에는 오차(MAPE 2.78%)가 있으므로, 최종 배합 결정은 작업자가 추천 결과를 검토 후 승인하여 적용하세요. 예측 품질 점수가 80점 이상이고 성분 합계가 100%일 때 적용을 권장합니다.

**Q2. 품질 점수 합계가 100%가 되지 않는다는 오류가 발생합니다.**
A. SN + AG + CU + PB + OTHER 항목의 합이 정확히 100%가 되어야 합니다. 각 항목을 다시 확인하고 소수점 반올림 오차가 있다면 OTHER 항목을 조정하여 합계를 맞추세요.

**Q3. 모델이 없다는 오류가 나타납니다.**
A. IT팀에 모델 학습을 요청하세요. 관리자는 `python scripts/train.py --model gradient_boosting` 명령으로 모델을 생성할 수 있습니다.

**Q4. SUP_B를 선택했을 때 추천 결과가 달라지는 이유는?**
A. 각 공급사의 원재료 성분 편차가 다르게 반영됩니다. SUP_B는 기준 공급사 대비 상대 편차를 반영하여 최적 배합비율이 조정됩니다. 실제 사용 공급사를 정확하게 선택하세요.

**Q5. DOE 시뮬레이터와 배합비율 추천의 차이는 무엇인가요?**
A. 배합비율 추천은 단일 공정 조건에서 최적 배합을 즉시 추천합니다. DOE 시뮬레이터는 여러 인자 조합을 체계적으로 실험하여 인자 간 상호작용을 분석하고 전반적인 최적 조건을 찾는 데 활용합니다. 신규 공정 개발이나 공정 개선 시 DOE를 활용하세요.

**Q6. 데이터 분석(EDA) 화면에서 데이터가 표시되지 않습니다.**
A. 학습 데이터 파일(`data/raw/formulation_history.csv`)이 존재하는지 확인하세요. 없으면 `python data/raw/generate_sample.py`로 샘플 데이터를 생성하거나 IT팀에 실데이터 경로를 문의하세요.

**Q7. 예측 품질 점수가 항상 비슷한 값이 나옵니다.**
A. Ridge 모델 선택 시 나타날 수 있는 현상입니다. gradient_boosting 또는 random_forest 모델을 선택하세요. 또한 입력 성분 비율이 목표값(SN 62%, AG 3%, CU 0.5%)에 가까울수록 높은 점수가 예측됩니다.

**Q8. 모바일(스마트폰)에서도 사용할 수 있나요?**
A. 네, 반응형 UI를 지원하여 스마트폰과 태블릿에서도 사용 가능합니다. Wi-Fi 환경에서 `http://서버IP:3000`으로 접속하세요. 화면이 작은 경우 가로 모드(landscape)를 권장합니다.

**Q9. 배합비율 추천 결과를 저장하거나 인쇄할 수 있나요?**
A. 현재 버전에서는 화면 캡처 또는 브라우저 인쇄(Ctrl+P) 기능을 활용하세요. 차기 버전에서 PDF 저장 및 이력 조회 기능이 추가될 예정입니다.

**Q10. 새로운 공급사가 추가되면 어떻게 해야 하나요?**
A. IT팀 또는 ML팀에 신규 공급사 등록을 요청하세요. `src/features/engineering.py`의 SUPPLIER_EFFECTS에 편차 값을 추가하고 모델을 재학습해야 합니다.

**Q11. R² 값이 0.627로 낮은데 시스템을 신뢰할 수 있나요?**
A. 현재 R² 값은 샘플 데이터의 노이즈(표준편차 3) 때문입니다. MAPE 2.78%는 실제 품질 점수의 ±2.78% 오차를 의미하며, 80점 예측 시 실제값은 77.8~82.2점 범위입니다. 실제 공정 데이터로 재학습 시 R² 0.85 이상 달성이 가능합니다.

**Q12. DOE Full Factorial 선택 시 실험 수가 너무 많아집니다.**
A. 인자가 3개 이상이고 수준이 3 이상이면 실험 수가 급격히 증가합니다 (3인자 3수준 = 27개). 이 경우 Fractional Factorial이나 LHS 방법으로 실험 수를 줄이세요. ML 시뮬레이션이므로 실험 비용은 없지만, 분석 결과의 명확성을 위해 20개 이하로 설계를 권장합니다.

**Q13. 여러 사람이 동시에 사용해도 되나요?**
A. 네, 동시 사용자 10명까지 정상 테스트 완료되었습니다. 다만, DOE 시뮬레이션처럼 연산이 많은 기능을 여러 명이 동시에 실행하면 응답 속도가 다소 느려질 수 있습니다.
