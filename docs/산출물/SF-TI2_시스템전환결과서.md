## 1. 문서 개요

본 문서는 납땜 합금(솔더) 배합 공정에서 숙련 작업자 경험 기반의 수기 운영 방식을 ML 자동화 시스템(Formulation ML)으로 전환한 결과를 기록한 시스템 전환 결과서이다.

전환 전 시스템의 데이터 이행 현황, 기능 전환 결과, 성능 비교, 검증 결과 및 최종 전환 완료 선언을 포함한다. 본 문서는 스마트공장 보급·확산사업(선도형 제조혁신 지원사업) 산출물로 활용된다.

**문서 범위**: 배합비율 추천, 품질 예측, EDA, 모델 현황, DOE 시뮬레이터 5개 핵심 기능 전환 전 과정

## 2. 전환 개요

### 2.1 전환 배경 (수기/경험 기반 → ML 자동화)

납땜 합금 배합 공정은 SN(주석), AG(은), CU(구리), PB(납) 4개 성분의 정밀한 배합비율이 제품 품질을 결정하는 핵심 공정이다. 전환 전 시스템에는 다음과 같은 구조적 문제점이 존재하였다.

| 문제 유형 | 상세 내용 | 영향 |
|-----------|-----------|------|
| 숙련 의존성 | 배합비율이 숙련 작업자 개인 경험에만 의존 | 작업자 부재 시 공정 중단 위험 |
| 편차 대응 부재 | 성분 편차 발생 시 수동 보정으로 즉각 대응 불가 | 불량률 증가, 품질 불안정 |
| 예측 불가 | 배합 후 품질 점수 예측 불가능 | 불량 발생 후 사후 처리 |
| DOE 비효율 | 실험계획법 설계에 수일 소요 | 신규 배합 개발 지연 |
| 지식 전수 불가 | 숙련자 노하우 문서화 부재 | 조직 기술력 단절 위험 |

위 문제를 해결하기 위해 300 LOT 이력 데이터 기반 ML 모델을 도입하여 배합비율 자동 추천, 품질 점수 예측, DOE 시뮬레이션을 자동화하는 Formulation ML 시스템으로 전환을 결정하였다.

**전환 대상 공정**: 납땜 합금(솔더) 배합 공정 — SN/AG/CU/PB 성분 배합비율 최적화

**전환 대상 기능**:

| 기능 ID | 기능명 | 전환 전 방식 | 전환 후 방식 |
|---------|--------|-------------|-------------|
| F-01 | 배합비율 결정 | 숙련 작업자 경험 판단 | SLSQP 최적화 알고리즘 자동 추천 |
| F-02 | 품질 예측 | 배합 후 실측 (사후) | ML 모델 사전 예측 (사전) |
| F-03 | 성분 분포 분석 | 엑셀 수작업 집계 | EDA 대시보드 자동 시각화 |
| F-04 | 모델 성능 관리 | 해당 없음 | MAE/RMSE/R²/MAPE 자동 산출 |
| F-05 | DOE 실험 설계 | 수작업 (수일 소요) | 6종 설계 방법 자동 생성 + ML 시뮬레이션 |

**전환 제외 범위**: 생산 실행 시스템(MES) 연동, ERP 연동 (차기 단계 계획)

## 3. 전환 전략

### 3.1 병행 운영 계획 (구 방식과 신 시스템 동시 운영 기간)

전환 리스크를 최소화하기 위해 2026-06-08부터 2026-06-14까지 7일간 기존 수기 방식과 신 ML 시스템을 병행 운영하였다.

**병행 운영 방식**:
- 제조팀 작업자가 기존 방식으로 배합비율 결정 → 동시에 ML 시스템으로 추천 결과 생성
- 양측 결과를 품질팀이 비교 기록하여 ML 시스템의 신뢰도 검증
- 불일치 케이스 발생 시 원인 분석 후 ML 모델 파라미터 조정

**병행 운영 기준**:

| 검증 항목 | 기준치 | 비고 |
|-----------|--------|------|
| 배합비율 추천 편차 | ±2% 이내 (SN 기준) | ML 추천 vs 작업자 판단 |
| 품질 점수 예측 오차 | MAPE 5% 이내 | ML 예측 vs 실측값 |
| 불량 판정 일치율 | 90% 이상 | 75점 기준 |
| 시스템 응답 시간 | 5초 이내 | 배합 추천 API |

**병행 운영 결과**: 7일간 14 LOT 병행 검증 수행, 모든 기준 충족 확인

### 3.2 데이터 이행 계획

#### 기존 LOT 이력 데이터 300건 CSV 변환

기존 엑셀 기반 LOT 이력 데이터를 ML 시스템 입력 형식으로 변환하였다.

**원본 데이터 구조** (엑셀):
- LOT 번호, 작업일, 투입 SN/AG/CU/PB 중량(kg), 용해 온도(°C), 용해 시간(분), 납품처, 품질 검사 점수, 불량 여부

**변환 후 데이터 구조** (CSV):
```
lot_id, sn_pct, ag_pct, cu_pct, pb_pct, other_pct, melt_temp_c, melt_time_min, supplier_id, quality_score, is_defect
```

**변환 절차**:
1. 엑셀 원본 파일 백업 (원본 보존)
2. 중량(kg) → 비율(%) 변환: `sn_pct = sn_kg / total_kg * 100`
3. 납품처 코드 → supplier_id 매핑 (A사→SUP_A, B사→SUP_B, C사→SUP_C)
4. 결측값 처리: melt_temp_c 결측 → 공정 표준값 250°C 대입, 나머지 결측 LOT 제외
5. 합계 검증: `sn_pct + ag_pct + cu_pct + pb_pct + other_pct ≈ 100` (±0.1% 허용)
6. `data/raw/formulation_history.csv` 저장

#### 데이터 검증 절차

| 검증 항목 | 검증 방법 | 허용 기준 |
|-----------|-----------|-----------|
| 건수 일치 | 원본 ROW 수 vs CSV 건수 대조 | 100% 일치 (결측 제외 건수 명시) |
| 비율 합계 | 각 LOT sum(성분%) ≈ 100% | ±0.1% 이내 |
| 범위 이상치 | quality_score 범위 50~100 | 범위 외 값 0건 |
| 불량 기준 일치 | is_defect = (quality_score < 75) | 100% 일치 |
| 공급사 코드 | SUP_A/B/C 외 코드 없음 | 이상 코드 0건 |
