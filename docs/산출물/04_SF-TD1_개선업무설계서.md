## 1. 원재료 입고 업무 개선

### 1.1 AS-IS (현행)

```
[공급사]
  │ 원재료 납품
  ▼
[입고 담당자]
  │ ① 육안 검사
  │ ② 수기 기록 (종이 서류)
  ▼
[XRF 성분 분석]
  │ 분석 시간: 30~60분/LOT
  │ 결과: 프린터 출력
  ▼
[품질팀 판정]
  │ 담당자 직접 규격서와 비교
  │ 합격/불합격 수기 기입
  ▼
[Excel 입력]
  │ 담당자가 분석 결과 수기 입력
  │ 오류 발생 빈도: 월 3~5건
  ▼
[창고 입고 완료]
```

### 1.2 TO-BE (개선)

```
[공급사]
  │ 원재료 납품
  ▼
[입고 담당자]
  │ ① 시스템에서 입고 등록 (LOT ID 자동 부여)
  │ ② 성분 분석 결과 직접 입력 or XRF 연동 자동 수신
  ▼
[AI 시스템 자동 분석]
  │ ① 목표값 대비 편차 자동 계산
  │    - Sn 편차 = 실측 - 62.0%
  │    - Ag 편차 = 실측 - 3.0%
  │    - Cu 편차 = 실측 - 0.5%
  │ ② 합격/불합격 자동 판정 (기준값 대비)
  │ ③ 이상 편차 시 즉시 알림 발송
  ▼
[입고 완료 등록]
  │ 이력 DB 자동 저장
  │ 공급사별 성분 품질 통계 자동 업데이트
  ▼
[창고 시스템 연동]
  재고 자동 반영
```

### 1.3 기대 효과

| 항목 | AS-IS | TO-BE | 개선 |
|------|-------|-------|------|
| 입고 데이터 입력 시간 | 20분/건 | 5분/건 | -75% |
| 입력 오류 | 월 3~5건 | 0건 목표 | -100% |
| 성분 이상 감지 | 육안·수기 | 실시간 자동 | 즉시 알림 |
| 공급사 품질 비교 | 불가 | 자동 통계 | 신규 |

## 2. 배합비율 결정 업무 개선

### 2.1 AS-IS (현행)

```
[생산 지시]
  │
  ▼
[숙련 작업자 판단]
  │ ① 이전 LOT 결과 참고 (기억 또는 서류 확인)
  │ ② 규격서 목표값 확인
  │ ③ 공급사별 성분 편차 경험적 보정
  │    (비체계적, 담당자 편차 큼)
  ▼
[배합비율 결정]
  │ 결과: 수기 배합 지시서 작성
  ▼
[원재료 칭량 및 용해]
  │
  ▼
[배합 완료 성분 검사]
  │ XRF 분석 (30~60분)
  ▼
[합격] → [포장/출하]
[불합격] → [재처리 또는 폐기]
   (재처리율 6.3%, 폐기 손실 발생)
```

### 2.2 TO-BE (개선)

```
[생산 지시]
  │
  ▼
[AI 배합 최적화 시스템]
  │ ① 입력: 공정 조건 (온도, 시간), 공급사 선택
  │ ② 처리:
  │    - 공급사별 최근 성분 편차 데이터 로드
  │    - scipy SLSQP 최적화 실행
  │    - 제약 조건: Sn+Ag+Cu+Pb = 100%
  │    - 목적 함수: ML 모델 품질 예측 최대화
  │ ③ 출력: 최적 배합비율 (Sn/Ag/Cu/Pb %)
  │          예상 품질 점수
  ▼
[배합 추천 확인]
  │ 작업자/관리자 검토 후 승인
  │ (시스템 추천 ± 조정 가능)
  ▼
[원재료 칭량 및 용해]
  │ 최적 배합비율 기반 정밀 칭량
  ▼
[배합 완료 성분 검사]
  │ XRF 분석 결과 시스템 자동 등록
  ▼
[품질 예측 vs 실측 비교]
  │ AI 모델 정확도 자동 평가
  ▼
[합격] → [LOT 시스템 등록] → [포장/출하]
[불합격] → [원인 분석 (AI)] → [개선 배합비 재추천]
```

### 2.3 기대 효과

| 항목 | AS-IS | TO-BE | 개선 |
|------|-------|-------|------|
| 배합비율 결정 시간 | 30분 (숙련자) | 5분 (AI 추천) | -83% |
| 불량 LOT 사전 예측 | 불가 | 사전 예측 가능 | 신규 |
| 재처리율 | 6.3% | 3% 목표 | -52% |
| 숙련자 의존도 | 매우 높음 | 낮음 | 노하우 시스템화 |
| 수율 | 91.2% | 95% 목표 | +3.8%p |

## 3. 품질 관리 업무 개선

### 3.1 AS-IS (현행)

```
[LOT 생산 완료]
       │
       ▼
[XRF 성분 분석]
  소요: 30~60분
       │
       ▼
[품질팀 판정]
  담당자 규격서 수동 비교
       │
   합격 ┤ 불합격
       ▼       ▼
[성적서 발행]  [재처리 결정]
  (Excel 수기)  (원인 추적 불가)
       │
       ▼
[월별 품질 보고서]
  (Excel 수기 집계, 2~3일 소요)
```

### 3.2 TO-BE (개선)

```
[배합 전 — 사전 예측]
  │ ML 모델로 품질 점수 사전 예측
  │ 예측 점수 < 70점 → 배합비율 재검토 알림
  ▼
[LOT 생산 완료]
       │
       ▼
[XRF 분석 결과 시스템 자동 등록]
  │ (연동) 또는 담당자 직접 입력
  ▼
[자동 품질 판정]
  │ ① 성분 기준 자동 합격/불합격 판정
  │ ② ML 예측 점수 vs 실측 비교
  │ ③ 이상 LOT 즉시 알림
  ▼
[품질 이력 DB 자동 저장]
  │ LOT-성분-품질 연계 저장
  ▼
[실시간 KPI 대시보드 업데이트]
  │ 수율, 불량률, 품질 평균 자동 갱신
  ▼
[불합격 LOT]
  │ AI 원인 분석 → 개선 배합비 재추천
```

## 4. 출하 업무 개선

### 4.1 AS-IS (현행)

```
[영업팀 출하 요청]
  (전화/이메일)
       │
       ▼
[물류팀 LOT 선정]
  창고 수기 확인, FIFO 수동 적용
       │
       ▼
[품질 성적서 발행]
  Excel 수기 작성 (1~2시간)
       │
       ▼
[포장 및 출하]
       │
       ▼
[출하 기록]
  Excel 수기 입력
```

### 4.2 TO-BE (개선)

```
[영업팀 출하 요청]
  시스템에서 출하 오더 등록
       │
       ▼
[AI LOT 추천]
  고객 규격 기반 최적 LOT 자동 매칭
  FIFO + 품질 점수 기반 최적 선정
       │
       ▼
[품질 성적서 자동 생성]
  LOT 성분 데이터 기반 자동 발행 (5분)
       │
       ▼
[포장 및 출하]
  시스템에서 출하 완료 처리
       │
       ▼
[재고 자동 차감]
  [출하 이력 자동 저장]
  [KPI 자동 업데이트]
```

## 5. 시스템 아키텍처 개선 개요

### 5.1 AS-IS 시스템 구조

```
[사용자]
   │
   ▼
[MS Excel / 수기 서류]
   │ 데이터 분산, 수동 집계
   ▼
[구형 ERP (2018년)]
   │ 수주/출하만 관리
   ▼
[XRF 분석기]
   독립 PC, 연동 없음
```

### 5.2 TO-BE 시스템 구조

```
[사용자 (Chrome Browser)]
   │
   ▼
[Next.js 14 프론트엔드]
  - 44개 화면
  - TypeScript 타입 안전성
  - 실시간 대시보드
   │ REST API
   ▼
[FastAPI 백엔드]
  - RESTful API
  - ML 모델 서빙
  - 인증/권한 관리
   │
   ├── [ML 엔진]
   │    - scikit-learn (RF, GB, Ridge)
   │    - XGBoost
   │    - scipy SLSQP 최적화
   │
   └── [PostgreSQL 15]
        - 성분/품질/공정 데이터
        - 사용자/권한 데이터
        - ML 모델 이력
```
