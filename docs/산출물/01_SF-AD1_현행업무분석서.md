## 1. 기업 개요

### 1.1 기업 소개

| 항목 | 내용 |
|------|------|
| 기업명 | (주)고려솔더 |
| 업종 | 솔더 합금 제조업 |
| 소재지 | 경기도 안산시 단원구 원시동 |
| 설립연도 | 1998년 |
| 직원 수 | 약 120명 |
| 주요 제품 | Sn계 솔더 합금 (Sn-Ag-Cu-Pb 계열) |
| 주요 고객 | 전자부품 제조업체, PCB 제조업체 |
| 매출 규모 | 약 450억원 (2025년 기준) |

### 1.2 생산 제품

고려솔더는 전자 산업용 솔더 합금을 전문으로 생산하며, 주요 제품군은 다음과 같다.

| 제품군 | 규격 | 주요 성분 | 주요 용도 |
|--------|------|-----------|-----------|
| Sn62 솔더 | Sn62/Pb36/Ag2 | Sn 62%, Pb 36%, Ag 2% | 일반 전자부품 납땜 |
| Sn63 솔더 | Sn63/Pb37 | Sn 63%, Pb 37% | 共晶(공정) 솔더 |
| SAC305 솔더 | Sn/Ag3/Cu0.5 | Sn 96.5%, Ag 3%, Cu 0.5% | 무연 솔더 (RoHS 대응) |
| 저온 솔더 | Sn/Bi 계열 | Sn 42%, Bi 58% | 열민감 부품용 |

## 2. 현행 업무 분석

### 2.1 원재료 입고 업무

#### 2.1.1 현행 프로세스 (AS-IS)

```
[공급사]
  │
  ▼
[원재료 입고] ──▶ [육안 검사] ──▶ [성분 샘플링 분석]
                                         │
                              성분 합격   │   성분 불합격
                                 ▼        │       ▼
                           [입고 등록]    │   [반품/재협의]
                           (Excel 수기)   │
                                 │        │
                                 ▼
                           [창고 입고]
```

#### 2.1.2 현행 문제점

| 문제 | 내용 | 영향 |
|------|------|------|
| 수기 기록 의존 | 성분 데이터를 Excel에 수기 입력 | 입력 오류 발생 빈도 월 3~5건 |
| 이력 추적 불가 | LOT-공급사-성분 연계 데이터 없음 | 품질 불량 시 원인 추적 2~3일 소요 |
| 실시간 성분 모니터링 부재 | 샘플링 분석 결과 지연 (최대 4시간) | 불량 원재료 투입 위험 |
| 공급사별 성분 편차 분석 불가 | 공급사 데이터 개별 관리 | 최적 공급사 선정 기준 없음 |

### 2.2 배합 및 제조 업무

#### 2.2.1 현행 프로세스 (AS-IS)

```
[배합 지시서 발행]
  (생산계획팀 → 제조팀, 수기 전달)
       │
       ▼
[원재료 칭량]
  (작업자 경험 기반 배합비율 결정)
       │
       ▼
[용해 및 혼합]
  (용해로 온도: 약 250°C, 작업자 육안 판단)
       │
       ▼
[배합 완료 검사]
  (성분 분석기로 Sn/Ag/Cu/Pb 비율 측정)
       │
   합격 ┤ 불합격
       ▼       ▼
  [포장/출하] [재처리/폐기]
```

#### 2.2.2 현행 배합비율 결정 방식

| 구분 | 내용 | 문제점 |
|------|------|--------|
| 의사결정 주체 | 제조팀 숙련 작업자 (경력 5~20년) | 숙련자 퇴사 시 노하우 유실 위험 |
| 배합비율 근거 | 과거 경험, 납품 규격서 | 공급사별 성분 편차 미반영 |
| 보정 방식 | 전회 LOT 결과 참고, 육안 조정 | 비체계적, 개인 편차 큼 |
| 품질 예측 | 없음 (제조 후 검사로 확인) | 불량 LOT 사후 발견 |

#### 2.2.3 배합 품질 현황 (2025년 기준)

| 지표 | 현황 | 목표 |
|------|------|------|
| 수율 | 91.2% | 95% |
| 불량률 | 8.8% | 5% 이하 |
| LOT 재처리 비율 | 6.3% | 3% 이하 |
| 평균 품질 점수 | 82.1점 | 88점 이상 |
| 성분 목표 달성률 | Sn: 87%, Ag: 79%, Cu: 83% | 95% 이상 |

### 2.3 품질 관리 업무

#### 2.3.1 현행 품질 검사 흐름

```
[제조 완료 LOT]
       │
       ▼
[성분 분석 (XRF 분석기)]
  - 분석 시간: LOT당 30~60분
  - 측정 항목: Sn, Ag, Cu, Pb 비율
       │
       ▼
[품질 판정]
  기준: 규격서 상하한 내 포함 여부
  판정자: 품질관리팀 (2명)
       │
   합격 ┤ 불합격
       ▼       ▼
[성적서 발행]  [재처리/폐기 결정]
  (Excel)
```

#### 2.3.2 품질 데이터 관리 현황

| 항목 | 현황 | 문제점 |
|------|------|--------|
| 데이터 저장 | Excel 파일 (팀별 분산) | 데이터 불일치, 검색 불편 |
| 이력 보관 | 3년치 Excel (약 8,000 LOT) | 패턴 분석 불가 |
| 실시간 모니터링 | 없음 | 품질 이상 신속 대응 불가 |
| 통계 분석 | 월별 수기 집계 | 시간 소요 크고 오류 가능 |

### 2.4 포장·출하 업무

#### 2.4.1 현행 프로세스

```
[출하 지시]
  (영업팀 → 물류팀, 전화/이메일)
       │
       ▼
[LOT 선정] ──▶ [포장] ──▶ [성적서 첨부] ──▶ [출하]
  (FIFO 원칙,          (Excel 출력)
   창고 수기 확인)
```

#### 2.4.2 현행 문제점

| 문제 | 내용 |
|------|------|
| LOT 추적 불가 | 출하 후 고객 클레임 시 해당 LOT 성분 데이터 찾기 어려움 |
| 출하 검사 지연 | 성적서 수기 작성으로 출하 리드타임 증가 (평균 2시간) |
| 재고 연동 없음 | 출하 후 재고 수동 업데이트 |

### 3.3 네트워크 현황

```
[인터넷]
    │
  [방화벽]
    │
[사내 LAN (1Gbps)]
    ├── [사무동 스위치] ── PC 10대
    ├── [제조동 스위치] ── PC 3대, 설비 PC
    └── [품질동 스위치] ── PC 2대, XRF PC
```

- 인터넷 회선: 500Mbps (KT 기업용)
- 사내 WiFi: 일부 구역 (802.11ac)
- 클라우드 사용: 없음 (전부 on-premise)
