## 제1장 시스템 개요

### 1.1 시스템 소개
고려솔더 배합비율 최적화 AI 스마트공장 시스템은 솔더 합금 제조 공정에서 발생하는 원재료 성분 편차를 자동으로 분석하고, AI/ML 모델을 통해 최적 배합비율을 추천하는 지능형 제조 관리 시스템입니다.

### 1.2 주요 기능 요약

| 기능 | 설명 | 주요 사용자 |
|------|------|------------|
| AI 대시보드 | 생산·품질·설비·출하 현황 실시간 모니터링 | 전 직원 |
| 배합비율 최적화 | AI가 최적 Sn/Ag/Cu/Pb 비율 자동 추천 | 제조팀 |
| 품질 예측 | ML 모델로 배합 전 품질 점수 사전 예측 | 품질팀, 제조팀 |
| 성분 편차 분석 | 공급사별 성분 편차 시계열 분석 | 구매팀, 품질팀 |
| 입고관리 | 원재료 입고 데이터 통합 관리 | 구매팀 |
| 출하관리 | LOT별 출하 이력 및 클레임 관리 | 영업팀, 물류팀 |
| KPI 관리 | 수율·품질 KPI 목표 대비 현황 | 경영진, 팀장 |

### 1.3 접속 방법

1. PC에서 Chrome 브라우저를 실행합니다.
2. 주소창에 `https://koryo-solder.local`을 입력합니다.
3. 로그인 화면에서 부여받은 계정을 입력합니다.
4. 로그인 후 **생산 현황 대시보드**로 자동 이동합니다.

> **주의**: Internet Explorer, Safari는 지원하지 않습니다. Chrome 또는 Edge 최신 버전을 사용하세요.

## 제3장 주요 기능 사용 방법

### 3.1 AI 배합 최적화 사용 방법 ★

**경로**: 배합비율 최적화AI → 배합 최적화

**목적**: 공정 조건과 공급사 정보를 입력하면 AI가 최적 배합비율(Sn/Ag/Cu/Pb %)을 자동 계산합니다.

**사용 순서**:

```
① [용해 온도] 입력
   - 일반적 설정: 250°C
   - 범위: 200°C ~ 320°C

② [처리 시간] 입력
   - 단위: 분(min)
   - 일반적 설정: 45분

③ [공급사 선택] 드롭다운에서 선택
   - SUP_A, SUP_B, SUP_C 중 선택
   - 선택한 공급사의 최근 성분 편차가 자동 반영됩니다

④ [사용 모델] 선택
   - 권장: Gradient Boosting (RMSE 최저, R² 최고)

⑤ [최적 배합 추천] 버튼 클릭
   - AI가 최적 Sn/Ag/Cu/Pb 비율을 계산합니다 (약 3~5초)

⑥ 결과 확인
   - 최적 배합비율: Sn/Ag/Cu/Pb 각 비율 확인
   - 예상 품질 점수: AI 예측 점수 확인
   - 합격 기준(70점) 이상 여부 확인
```

> **TIP**: 공급사를 변경하면 해당 공급사의 성분 편차 특성이 배합 계산에 반영되어 최적 비율이 달라질 수 있습니다.

---

### 3.2 품질 예측 사용 방법

**경로**: 배합비율 최적화AI → 품질 예측

**목적**: 배합비율을 입력하면 실제 제조 전에 예상 품질 점수를 확인합니다.

**사용 순서**:

```
① 각 성분 비율 입력
   - Sn (주석): 슬라이더 또는 직접 입력 (단위: %)
   - Ag (은): 슬라이더 또는 직접 입력
   - Cu (구리): 슬라이더 또는 직접 입력
   - Pb (납): 슬라이더 또는 직접 입력
   - 합계: 반드시 100.0%가 되어야 합니다

② 사용할 ML 모델 선택
   - Gradient Boosting (권장)
   - Random Forest
   - XGBoost
   - Ridge

③ [예측하기] 버튼 클릭

④ 결과 확인
   - 예상 품질 점수 (0~100점)
   - 합격 여부 (70점 이상 합격)
   - 성분 편차 요약 (목표값 대비)
```

> **주의**: 합계가 정확히 100%가 아니면 예측 버튼이 비활성화됩니다.

---

### 3.3 성분 편차 분석 사용 방법

**경로**: 배합비율 최적화AI → 성분 편차 분석

**목적**: 최근 90일간 입고 원재료 및 생산 LOT의 성분 편차 추이를 분석합니다.

**화면 설명**:

| 섹션 | 내용 |
|------|------|
| Sn 편차 추이 | 90일간 Sn 실측값 - 목표값(62.0%)의 편차 라인 차트 |
| Ag 편차 추이 | 90일간 Ag 편차 (목표: 3.0%) |
| Cu 편차 추이 | 90일간 Cu 편차 (목표: 0.5%) |
| 공급사별 비교 | SUP_A/B/C 성분 안정성 비교표 |

> **판독 방법**: 편차선이 0에 가깝고 변동이 작을수록 안정적인 성분을 공급하는 것입니다. 편차가 ±2% 이상이면 공급사 협의 또는 배합비율 보정이 필요합니다.

---

### 3.4 입고 데이터 등록 방법

**경로**: 입고관리 → 성분 데이터

**사용 순서**:

```
① [+ 성분 데이터 등록] 버튼 클릭
② LOT ID 또는 입고 번호 입력
③ 측정 날짜 선택
④ 공급사 선택
⑤ XRF 분석 결과 입력
   - Sn (%): 소수점 1자리
   - Ag (%): 소수점 2자리
   - Cu (%): 소수점 2자리
   - Pb (%): 자동 계산 (100 - Sn - Ag - Cu)
⑥ [저장] 클릭
   - 목표값 대비 편차 자동 계산
   - 편차 초과 시 자동 경고 알림 발송
```

### 4.2 알림 임계값 설정

**경로**: 사용자/시스템관리 → 알림 설정

| 알림 항목 | 기본값 | 설명 |
|-----------|--------|------|
| 품질 불합격 기준 | 70점 미만 | 이 점수 미만이면 불합격 알림 |
| Sn 편차 경고 | ±2.0% | 목표(62%) 대비 ±2% 초과 시 경고 |
| Ag 편차 경고 | ±0.3% | 목표(3%) 대비 ±0.3% 초과 시 경고 |
| Cu 편차 경고 | ±0.1% | 목표(0.5%) 대비 ±0.1% 초과 시 경고 |
| 설비 온도 경고 | 255°C 초과 | 설비 온도 초과 시 경고 |

### 4.3 ML 모델 목표값 설정

**경로**: 사용자/시스템관리 → 시스템 설정

| 설정 항목 | 기본값 | 설명 |
|-----------|--------|------|
| SN_TARGET | 62.0 | Sn 목표 비율 (%) |
| AG_TARGET | 3.0 | Ag 목표 비율 (%) |
| CU_TARGET | 0.5 | Cu 목표 비율 (%) |
| 품질 합격 기준 | 70점 | 품질 점수 합격 최저선 |

> **주의**: 목표값 변경 시 ML 모델 재학습이 필요합니다. 변경 전 반드시 ML 담당자에게 확인하세요.

## 제5장 장애 처리

### 5.1 자주 발생하는 문제

| 증상 | 원인 | 조치 방법 |
|------|------|-----------|
| 로그인이 안 됨 | 비밀번호 오류 or 계정 잠금 | 관리자에게 비밀번호 초기화 요청 |
| 화면이 느리게 로딩됨 | 네트워크 불안정 | 페이지 새로고침 (F5), 네트워크 담당 연락 |
| 예측 결과가 안 나옴 | ML 서버 응답 없음 | 화면 새로고침 후 재시도, 지속 시 IT 담당 연락 |
| 배합 최적화가 실패함 | 입력 조건 범위 이탈 | 입력값 범위 재확인 (온도: 200~320°C) |
| 데이터가 조회되지 않음 | 세션 만료 (30분) | 로그아웃 후 재로그인 |
| 알림이 오지 않음 | 이메일 설정 오류 | 시스템 설정 → 알림 채널 확인 |

### 5.2 비상 연락처

## 제6장 데이터 백업 및 복구

### 6.1 자동 백업
- 매일 새벽 01:00 자동 전체 백업 실행
- 백업 위치: `/backup/postgres/` (별도 디스크)
- 보관 기간: 90일

### 6.2 데이터 내보내기 (사용자)
1. 데이터관리시스템 → 다운로드 이동
2. 기간/항목 선택 후 [CSV 다운로드] 클릭
3. 파일이 자동으로 PC에 저장됩니다

### 6.3 시스템 복구 절차 (IT 관리자)

```
① 장애 감지 → 비상 연락처에 즉시 보고
② Docker 서비스 상태 확인
   docker ps -a
③ 서비스 재시작
   docker compose restart
④ DB 연결 확인
   docker exec postgres-db psql -U koryo -c "\l"
⑤ 복구 불가 시 백업에서 복원
   pg_restore -d koryo_solder_db /backup/postgres/latest.dump
⑥ 서비스 정상화 확인 후 사용자 공지
```
