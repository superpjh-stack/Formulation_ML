## 1. 문서 개요

### 1.1 목적

본 문서는 Formulation ML 시스템의 프론트엔드 화면 구성, UI 컴포넌트, 사용자 인터랙션 흐름을 정의한다. 개발자, 디자이너, QA 담당자가 화면 구현 시 공통 기준으로 활용한다.

**주요 대상 독자**:
- 프론트엔드 개발자 (Next.js 14 구현)
- 백엔드 개발자 (API 연동 인터페이스 확인)
- QA 엔지니어 (화면 검증 기준)
- 현업 담당자 (요건 확인 및 검수)

### 1.2 화면 설계 원칙

#### 원칙 1 — 정보 우선순위 명확화

스마트공장 현장 작업자와 공정 엔지니어가 주 사용자다. 핵심 결과(추천 배합비율, 품질 점수, 불량 여부)를 화면 최상단에 배치하고, 상세 데이터는 하단 또는 접기 영역에 위치시킨다.

#### 원칙 2 — 단방향 조작 플로우

입력 → 실행 → 결과 순서의 단방향 흐름을 유지한다. 각 화면은 독립적으로 동작하며 탭 이동 없이 단일 화면 내에서 작업을 완료할 수 있도록 설계한다.

#### 원칙 3 — Tailwind CSS 기반 일관성

모든 스타일은 Tailwind CSS 유틸리티 클래스를 사용한다. 커스텀 CSS는 최소화하며, 디자인 토큰(색상, 간격, 타이포그래피)은 `tailwind.config.js`에서 중앙 관리한다.

#### 원칙 4 — 반응형 우선 설계

모바일(360px) 기준으로 설계 후 데스크톱(1280px+)으로 확장하는 Mobile-First 방식을 적용한다. Tailwind 브레이크포인트(`sm`, `md`, `lg`, `xl`)를 일관되게 사용한다.

#### 원칙 5 — 접근성(A11y) 준수

WCAG 2.1 AA 기준을 준수한다. 모든 인터랙티브 요소에 키보드 접근성을 보장하고, 색상 대비비 4.5:1 이상을 유지하며, 스크린 리더 대응을 위한 `aria-label` 및 `role` 속성을 적용한다.

#### 원칙 6 — 로딩 상태 명시

ML 추론 API는 응답 지연(1~5초)이 발생할 수 있다. 모든 비동기 요청에 로딩 스피너 및 스켈레톤 UI를 적용하여 사용자 불확실성을 제거한다.

| `shadow-md` | 강조 카드, 드롭다운 |
| `shadow-lg` | 모달, 팝오버 |

#### 입력 필드 정의

#### 설계 방법 정의

| 방법 | 코드 | 설명 | 적합 상황 |
|------|------|------|-----------|
| Full Factorial | `full_factorial` | 모든 요인 수준 조합 | 요인 수 ≤ 3, 완전한 정보 필요 |
| Fractional Factorial | `fractional_factorial` | 전체 조합의 일부 | 요인 수 4~7, 스크리닝 |
| CCD (중심합성설계) | `ccd` | 2차 반응표면 모델 | 최적 조건 정밀 탐색 |
| Box-Behnken | `box_behnken` | 꼭짓점 제외 설계 | 3요인 이상, 코너 실험 불가 시 |
| Taguchi | `taguchi` | 직교배열표 기반 | 품질 강건 설계 |
| LHS (라틴 초정방체 표본추출) | `lhs` | 균등 확률 공간 표본 | 컴퓨터 실험, 연속 공간 탐색 |

| `sm` | 640px | 모바일 가로, 소형 태블릿 |
| `md` | 768px | 태블릿 |
| `lg` | 1024px | 소형 노트북 |
| `xl` | 1280px | 데스크톱 |
| `2xl` | 1536px | 대형 모니터 |

### 5.2 레이아웃 변화 상세

#### SCR-001 홈 화면

| 화면 크기 | 카드 레이아웃 |
|-----------|---------------|
| 모바일 | 1열 (`grid-cols-1`) |
| sm | 2열 (`sm:grid-cols-2`) |
| lg | 3열 (`lg:grid-cols-3`) |
| DOE 카드 | 모바일: 1열, sm+: 2열 span (`sm:col-span-2`) |

#### SCR-002 배합비율 추천

| 화면 크기 | 레이아웃 |
|-----------|----------|
| 모바일 | 입력 폼(상) → 결과(하) 수직 적층 |
| md+ | 입력 폼(좌, 40%) + 결과(우, 60%) 나란히 |

#### SCR-006 DOE 시뮬레이터

| 화면 크기 | 탭 표시 방식 |
|-----------|--------------|
| 모바일 | 탭 텍스트 단축 (1단계, 2단계, 3단계, 4단계) |
| md+ | 전체 탭명 표시 (설계, 시뮬레이션, 분석, 최적화) |

### 5.3 터치 인터페이스 고려사항

- 모든 터치 타깃 최소 크기: 44x44px (`min-h-[44px] min-w-[44px]`)
- 슬라이더 입력은 모바일에서 `input[type=range]`의 thumb 크기를 24px 이상으로 확대
- 테이블은 모바일에서 `overflow-x: auto` 컨테이너로 감싸 가로 스크롤 허용
